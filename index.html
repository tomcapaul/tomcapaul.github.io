Secure Coding Principles

  • This course covers code security topics I feel are critical for all students majoring in any form of computer science.  From my work as a consultant, I saw a severe lack of fundamental coding skills with regards to code security in industry.  I felt this was something that could and should be addressed at the academic level.  I was able to spend a quarter at Microsoft on sabbatical understudying one of the top members of their Secure Windows Initiative (SWI) team, Michael Howard.  His book with David LeBlanc, Writing Secure Code, was a foundation piece for the class I ended up building. The class itself surveys a variety of code security topics that most developers will encounter or should be aware of in their careers: buffer overflow, integer/numeric overflow, password security, threat modeling, basic key exchange principles, command injection, SQL injection, cross site scripting attacks, cross site request forgery attacks, input validation via regular expressions, principle of least privilege, race conditions, and much more.  Students are asked to employ the previous list of ideas in programming solutions with code security in mind.  Students even work in teams and attack one-another's code.
